Sage example 4. Structure of unit group modulo \(n\).

Once we have created a modular ring R=Integers(n), we can explore the structure of the unit group \(R^*\) using G=R.unit_groups().
🔗
Notice that the description of \(G\) is not precisely what we expected. Sage is describing \(G\) abstractly as a cyclic group of cardinality \(16\text{.}\) It has obscured the origins of this group as a subset of \(R\text{.}\) Let’s see what the elements of \(G\) look like in Sage.
🔗
It appears Sage has realized \(G\) as the cyclic group generated by an element it calls \(f\text{.}\) Sage has not actually forgotten the humble beginnings of \(f\text{,}\) as the method f.value() reveals.
🔗
Aha! We now see that \(f\) corresponds to the element \(3\) in \(R\text{.}\) The Sage group object \(G\) has additional useful methods. For example, \(G.subgroups()\) produces a list of all subgroups of \(G\text{,}\) which in our language is the list of the distinct cyclic subgroups \(\angvec{r}\) as we let \(r\) vary over \((\Z/17\Z)^*\text{.}\)
🔗
Note that once again, Sage describes these subgroups in terms of the element \(f\text{.}\) To reminder ourselves of what \(f\) is here, we can use the .gen() method, followed by the .value() method. For the list comprehension below, we need to exclude the trivial subgroup at the end. Do you see how we did this?
